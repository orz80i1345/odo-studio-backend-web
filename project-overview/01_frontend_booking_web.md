# booking-web 前台預約網站需求

## 目的

前台是給一般使用者看的攝影棚網站，用來：

- 展示品牌形象
- 展示空間與佈景
- 說明價格與規範
- 讓使用者 24H 線上預約
- 顯示匯款資訊
- 查看自己的預約狀態

## 前台頁面

- 首頁
- 空間介紹
- 佈景展示
- 佈景詳細
- 價格方案
- 登入
- 註冊
- 預約月曆
- 預約確認
- 預約完成 / 待付款
- 我的預約
- 預約詳細
- FAQ
- 聯絡資訊
- 隱私權政策
- 使用條款

## 首頁內容

首頁不要做成一般 landing page。

區塊建議：

1. Opening section
2. Studio mood gallery
3. About Ode Studio
4. Scene preview
5. Price preview
6. Booking preview
7. Notes
8. Footer

風格：

- 像生活風格雜誌開場
- 大量留白
- 圖片像作品集，不只是背景
- CTA 清楚但安靜

## 空間介紹

內容包含：

- 攝影棚名稱
- 空間介紹
- 地址區域描述
- 自然光特色
- 空間坪數
- 可容納人數
- 適合拍攝類型
- 設備資訊
- 場地規範
- 注意事項
- 圖片 gallery

適合拍攝類型：

- 人像寫真
- 品牌形象
- 商品攝影
- 生活風格照
- 短影音
- 小型訪談
- 社群內容拍攝

## 佈景展示

每個佈景包含：

- 佈景名稱
- 簡短介紹
- 詳細描述
- 封面圖片
- 多張圖片
- 適合拍攝類型
- 是否顯示在前台
- 排序

呈現方式：

- 不要像商品列表
- 要像 portfolio / editorial gallery
- 圖片比例可混合直式與橫式
- Hover 狀態要細膩

## 價格方案

需要支援：

- 小時計費
- 半日方案
- 全日方案
- 特殊時段
- 未來加購項目

每個價格方案：

- 名稱
- 說明
- 時長
- 價格
- 是否啟用
- 排序

設計要求：

- 不要像 SaaS pricing cards
- 不要過度促銷
- 資訊清楚、安靜、有信任感

## 會員與登入

使用者帳號欄位：

- Email
- 姓名
- 電話
- 密碼

登入方式：

- Email / password
- 未來可擴充 Google Login / LINE Login

預約頁面限制：

- 預約頁面必須登入後才能進入
- 未登入點擊「預約」時，要跳到登入頁
- 登入成功後，導回原本要去的預約頁

登入後預約表單自動帶入：

- Email
- 姓名
- 電話

## 預約流程

完整流程：

1. 使用者進入網站
2. 查看首頁、空間介紹、佈景、價格
3. 點擊預約
4. 系統檢查是否登入
5. 未登入：跳轉登入頁
6. 登入成功後回到預約頁
7. 使用者選擇月份
8. 使用者選擇日期
9. 使用者選擇時段
10. 系統顯示預約摘要
11. 自動帶入姓名、Email、電話
12. 使用者確認日期與時段
13. 例如：07/03 14:00-19:00
14. 使用者點擊完成
15. 建立 booking
16. booking status = `pending_payment`
17. payment status = `unpaid` 或 `pending`
18. 顯示匯款帳號與付款資訊
19. 寄送預約確認信
20. 後台確認付款
21. booking status = `confirmed`
22. 預約日前一天寄送電子鎖密碼
23. 預約結束後可改成 `completed`

## 預約月曆

預約畫面使用「每月月曆」。

功能：

- 顯示當月月曆
- 可切換上個月 / 下個月
- 每天顯示可預約狀態
- 點擊日期後顯示當天可預約時段
- 狀態色清楚但安靜

日期狀態：

- 可預約
- 部分時段可預約
- 已滿
- 公休
- 維護
- 包場
- 尚未開放

## 預約完成 / 待付款頁

點擊完成後顯示匯款帳號，而不是付款完成。

顯示：

- 預約編號
- 預約日期
- 預約時段
- 預約人姓名
- Email
- 電話
- 匯款金額
- 匯款銀行
- 銀行代碼
- 戶名
- 帳號
- 匯款期限
- 注意事項
- 後台確認款項後才算預約完成
- 未來會串接綠界金流
